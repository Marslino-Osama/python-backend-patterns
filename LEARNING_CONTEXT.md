# Learning Context & Philosophy

## Primary Objective

Master Python backend development and distributed systems design through practical implementation, not passive learning.

## What Success Looks Like

- Build production-grade reusable Python backend modules.
- Understand backend architecture tradeoffs from implementation experience.
- Create a portfolio that demonstrates senior-level backend engineering.
- Develop muscle memory for Python typing, async, validation, testing, and clean architecture.
- Gain hands-on experience with scalability and reliability patterns.

## Student Profile

- Background: software engineering and programming fundamentals.
- Already understands: variables, loops, functions, algorithms, debugging basics, Git basics.
- Learning focus: Python backend ecosystem, clean architecture, async patterns, distributed systems implementation.

## Core Principles

### 1. Real-World Complexity from Day One

Every project should use realistic structure: configuration, validation, error handling, database access, tests, Docker, and clear module boundaries.

### 2. Build Reusable Modules

Foundation projects should become modules that can be imported into later projects.

### 3. Framework-Agnostic Patterns

FastAPI is the primary framework because it is modern, typed, and productive. Flask is used as an alternative implementation to prove that the architecture does not depend on one framework.

### 4. Progressive Complexity

- Phase 1: standalone modules.
- Phase 2: scaling and composition.
- Phase 3: reliability and advanced architecture.
- Capstone: production API in FastAPI and Flask.

## Learning Flow

```text
PLAN -> BUILD -> UNDERSTAND -> REFACTOR -> EXTRACT -> DOCUMENT
```

## Anti-Patterns to Avoid

- Copy-pasting without understanding.
- Building everything at once.
- Ignoring errors and logs.
- Over-engineering before the problem requires it.
- Skipping tests.
- Hiding important dependencies behind magic abstractions.
